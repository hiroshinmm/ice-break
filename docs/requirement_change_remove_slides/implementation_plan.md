# スライド生成廃止とWeb画像利用への変更計画

## 概要
スライド生成（PuppeteerによるHTMLから画像への変換）を廃止し、記事のOGP画像（Web取得画像）を直接使用することで、システムの軽量化とビルド時間の短縮を図ります。

## ユーザーレビューが必要な事項
- スライド（1920x1080）形式での表示がなくなるため、Webギャラリーおよびメール通知のレイアウトが記事画像+テキストの形式に変更されます。
- 画像の「軽量化」については、ダウンロードしてリサイズ・圧縮を行うか、あるいは単に重い画像を読み込まないようにするかの方針。

## 修理・変更内容

### 1. ドキュメントの更新
- `docs/requirements_specification.md` の修正。
- `docs/final_requirements_specification/requirements_specification.md` の修正。

### 2. システム構成の変更
- `src/captureSlides.js` の役割を大幅に縮小（または廃止し、ギャラリー生成機能を分離）。
- `src/templates/index.ejs` の修正（スライドPNGの代わりに記事画像を表示）。
- `src/sendNotification.js` の修正（JPG添付の代わりに、HTMLメール内で記事画像を参照）。

### 3. バッチ処理（GitHub Actions）の更新
- スライド生成ステップの削除。

## 確認計画
- ローカルビルドを実行し、`index.html` が記事画像を用いて正しく表示されることを確認。
- メール送信テストを行い、記事画像が含まれることを確認。
