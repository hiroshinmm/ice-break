# デイリー・テックブリーフィング システム要求仕様書 (完全版)

## 1. プロジェクト概要
本システムは、ディスプレイ開発エンジニアや技術愛好家をターゲットに、最先端技術（AI, XR, Display等）のニュースを自動で収集・分析・視覚化し、毎朝のルーチンとして提供する自動化ソリューションです。

## 2. 業務フロー
1.  **ニュース収集**: 毎朝、RSS/Google Newsから10カテゴリの最新ニュースを取得。
2.  **AI分析**: Gemini APIを活用し、要約とエンジニア向けの独自インサイトを生成。
3.  **ビジュアル取得**: 記事本来のイメージ画像（OGP等）を取得。画像がない場合は共通のデフォルトアイコンを表示。
4.  **メディア出力**: Puppeteerを用いて、HTMLテンプレートを画像（PNG/JPG）として出力。
5.  **配信**: Webサイト（GitHub Pages）の更新と、関係者へのメール送信。

## 3. 技術仕様 (Technical Specifications)

### 3.1 AI インテリジェンス
- **LLM**: Google Gemini API (`gemini-2.0-flash-lite-preview-02-05`)
  - **役割**: 記事抽出、150文字要約、技術インサイト生成。
- **イメージ画像ポリシー**: 
  - 記事本来のイメージ画像を優先使用。
  - 画像欠落時はローカルのデフォルトニュースアイコン（`src/assets/default_news.png`）を使用し、視覚的一貫性を維持。
  - 安定性と速度を優先し、AIによる動的画像生成は廃止。

### 3.2 フロントエンド・デザイン
- **レイアウト構成**: 
  - **Container**: 最大幅 1800px のワイド設計。
  - **Grid**: PC = 2列, モバイル(<1100px) = 1列。
  - **Typography**: Google Fonts "Outfit" を使用したモダンなタイポグラフィ。
- **ユーザー体験**: 
  - 記事タイトルがオリジナルソースへの直接リンクとして機能。
  - ダークモード/ライトモードに対応した洗練された配色（グラデーション背景等）。

### 3.3 バックエンド・自動化
- **環境**: Node.js & Puppeteer
- **安定化処理**:
  - **Timeout**: 以前の 90秒から標準的な **30秒** に戻し、ビルド時間を大幅に短縮。
  - **Wait**: `networkidle2` による安定した読み込み待機。
- **日付処理**: `Intl.DateTimeFormat` を用い、`Asia/Tokyo` タイムゾーンで全ての出力を統一。

### 3.4 運用・通知
- **GitHub Actions**:
  - `cron: '0 23 * * *'` (JST 8:00 AM) に毎日実行。
  - Secrets管理: `GEMINI_API_KEY`, `GMAIL_USER`, `GMAIL_PASS` 等。
- **メール配信**: 
  - メーラーでのレンダリングを考慮し、Flexbox ではなく固定レイアウトに近い安全な構成を採用。
  - 1920x1080 画像の軽量JPG（品質60%）を添付。

## 4. カテゴリ定義
1. **AI**: 生成AI、エージェント、基盤モデル
2. **SRD / XR**: 空間再現ディスプレイ、VR/AR/MR
3. **Gaming Monitor**: 高リフレッシュレート、HDR
4. **Production Monitor**: 放送用、映像制作向け
5. **Camera Control**: カメラ制御技術、イメージセンサー
6. **Projector**: レーザー光源、光学エンジン
7. **LED Wall Display**: 大型LED、サイネージ
8. **TV**: コンシューマー向けテレビ、パネル技術
9. **SONY**: ソニー関連、Xperia、α
10. **TCL**: パネルメーカー、最新ディスプレイ

## 5. 品質・制約事項
- **安定性**: 外部生成に依存しないローカルリソースの活用により、GitHub Actions でのタイムアウトを根本的に解消。
- **保守性**: EJSテンプレートによる構造とデザインの分離。
- **パフォーマンス**: AI画像生成の廃止により、1回の実行時間を数分単位で削減。
