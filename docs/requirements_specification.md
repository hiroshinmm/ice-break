# デイリー・テックブリーフィング システム要件定義書

## 1. 概要
本システムは、特定の技術カテゴリ（AI、XR、ディスプレイ技術、カメラ制御等）の最新ニュースを自動収集し、Gemini APIを用いて技術的な要約とインサイトを生成します。その内容を AI 画像生成技術（Pollinations.ai）と統合してスライド画像化し、Webギャラリーの更新および毎朝のメール通知を行う自動化システムです。

## 2. システム構成
- **Frontend**: HTML5, Vanilla CSS, EJS (テンプレートエンジン)
- **Backend**: Node.js
- **Automation**: GitHub Actions (CI/CD)
- **AI (LLM)**: Google Gemini API (gemini-3.1-flash-lite-preview-02-05 等)
- **Notification**: Nodemailer (Gmail SMTP)
- **Deployment**: GitHub Pages

## 3. 主要機能と詳細要件

### 3.1 ニュース収集 (fetchNews)
- **対象カテゴリ**: AI, SRD/XR, Gaming Monitor, Production Monitor, Camera Control, Projector, LED Wall Display, TV, SONY, TCL の全10カテゴリ。
- **取得ソース**: RSSフィードおよびGoogle News検索。
- **フィルタリング**: 過去7日以内のニュースを優先し、重複排除を行った上で各カテゴリの最新情報を維持。

### 3.2 洞察生成 (generateInsights)
- **ニュース要約**: 各カテゴリから最も重要なトピックを1つ選定し、約150文字の日本語で要約。
- **技術インサイト**: ディスプレイ・ソフトウェアエンジニア向けの専門的な考察（技術的インプリケーション）を生成。

### 3.3 コンテンツ準備
- **イメージ画像取得**: 
    - 記事本来のイメージ画像（OGP等）を取得し、Webギャラリーおよびメール通知に使用。
    - 画像は表示パフォーマンスを優先し、軽量なものを採用する。
    - 画像欠落時はローカルのデフォルトニュースアイコンを使用。
- **JST対応**: 日付表示は常に日本時間（Asia/Tokyo）に固定。

### 3.4 Webギャラリー (index.html)
- **デザイン**: 
    - **コンテナ幅**: 最大 1800px のワイドレイアウト。
    - **グリッド構成**: PC閲覧時は 2カラム、画面幅 1100px 以下のモバイル閲覧時は 1カラムに自動切り替え。
    - **フォント**: モダンで読みやすい「Outfit」フォントを採用。
- **アクセシビリティ**: 記事タイトルをクリックすることで、オリジナルのニュースソースへ直接アクセス可能。
- **表示画像**: スライド画像の代わりに、各ニュース記事から抽出したオリジナルの画像を直接表示。

### 3.5 通知機能 (sendNotification)
- **自動実行スケジュール**: 毎日 日本時間(JST) 朝8:00。
- **メール通知**: 
    - 1カラムのモバイル最適化レイアウト。
    - メールの件名および本文の日付を日本時間で正確に表示。
    - 各記事の要約・インサイトと共に、抽出されたオリジナルの画像を軽量な形式で表示。

## 4. プロジェクトの進化と改善履歴
- **構成の最適化（最新）**: ビルド時間短縮と安定性向上のため、スライド画像（Puppeteer）およびAI画像生成を廃止し、Web取得画像の直接利用へ移行。
- **画像エンジンの刷新**: 固定IDマッピング（初期） → キーワード検索（中期） → 記事特化型AI画像生成 → オリジナル記事画像（現在）へと進化。
- **レイアウトの拡張**: 1カラム（初期）から大型モニターに対応した 2カラム（1800px幅）へ。
- **時刻・日付の厳密化**: GitHub Actions（UTC環境）でも常に日本時間で動作・表示されるよう完全修正。
- **ユーザー体験の向上**: 元記事へのリンク、技術インサイトのメール直接記述、フォントの可読性向上。

## 5. 今後の拡張性
- **動画抽出**: ニュースに含まれる動画コンテンツの自動キャプチャ。
- **多言語対応**: インサイトの英語/日本語切り替え機能。
- **Slack/Teams通知**: 配信先のマルチプラットフォーム化。
