# タスクリスト: アーカイブ機能（過去ログ閲覧）の実装

## 概要
システムの情報を蓄積し、過去のブリーフィングを Web ギャラリーで閲覧できるようにする。また、リポジトリの肥大化を防ぐための自動クリーンアップ機能を実装する。

## タスク一覧

### フェーズ 1: データ永続化の基盤 (Data Infrastructure)
- [ ] **アーカイブフォルダ構造の作成**
  - `data/archives/YYYY/MM/DD.json` 形式での保存ロジック実装
- [ ] **GitHub Actions の自動コミット設定**
  - 新規 JSON ファイルをリポジトリに自動 push するワークフローの追加
- [ ] **Gemini 分析ロジックの調整**
  - 保存される JSON に「日付情報」を明示的に含める

### フェーズ 2: Web ギャラリーの拡張 (Frontend)
- [ ] **日付選択 UI の追加**
  - ヘッダーへの日付セレクター（またはドロップダウン）の設置
- [ ] **動的データ読み込み (Lazy Load)**
  - URL パラメータ（`?date=YYYY-MM-DD`）による表示切り替え機能
- [ ] **ナビゲーション強化**
  - 「前日の記事」「翌日の記事」ボタンの実装

### フェーズ 3: リソース管理と最適化 (Maintenance)
- [ ] **自動画像クリーンアップの実装**
  - 14日以上経過した画像をリポジトリから自動削除するロジック
- [ ] **フォールバック処理**
  - ローカル画像がない場合にオリジナル URL を参照する仕組み (Frontend)

### フェーズ 4: 通知の連携 (Notification)
- [ ] **メール内アーカイブリンクの追加**
  - 今日の記事だけでなく、アーカイブ一覧への誘導リンクを設置
